# Skills Overview

## Purpose

This is the skill index — read it **first** whenever a practitioner describes a task, to identify the single most relevant skill before loading anything else. Match the task to one skill, then load only that skill file in full, which keeps the context window efficient rather than pulling every skill into memory at once.

## Skill types

Every skill in the OS is one of four kinds. The taxonomy helps you reason about *what a skill is for* and *what should run before it*:

- **Collectors** — gather context before the work starts: account data, business background, competitor intelligence. Everything else runs better when a Collector has run first.
- **Diagnostics** — identify what is wrong and why. They surface issues, prioritise them, and point toward the fix. They never touch a live account.
- **Builders** — create assets: ad copy, report narratives, client briefs, onboarding documents. Grounded in real context, not generic output.
- **Reviews** — assess ongoing performance: campaign health, search term quality, monthly results. Recurring, fast, decision-ready.

> **Collectors are on the roadmap.** The first will be a **Client Context Collector** that interviews the practitioner and populates the Obsidian client file — the Collector that feeds every other skill.

## How to use it

- Read this index first when the practitioner describes a task.
- Match the task description to the most relevant skill below.
- Load only the matched skill file in full before proceeding.
- If no skill matches, use the knowledge files directly and proceed without a skill.

## Skill index

- **Account Audit** *(Diagnostic)* — `skills/account-audit.md` — Structured diagnostic across five dimensions: conversion tracking, structure, bidding, creative, search term quality. *Reach for it when* onboarding a new client, running a health check, or diagnosing a performance drop.
- **Campaign Review** *(Review)* — `skills/campaign-review.md` — Fast weekly or fortnightly campaign performance review. *Reach for it when* checking in on active campaigns, preparing for a client call, or after a budget or target change.
- **Monthly Report Brief** *(Builder)* — `skills/monthly-report-brief.md` — Generates the written narrative for a monthly client report. *Reach for it when* producing monthly reporting, preparing for a performance review, or writing a client summary.
- **Search Term Analysis** *(Diagnostic)* — `skills/search-term-analysis.md` — Analyses search term reports with conversion lag offsets, flags waste, promotes converting terms, and outputs importable negative keyword lists. *Reach for it when* running weekly search term hygiene, after launching broad match campaigns, or when CPA has risen unexpectedly.
